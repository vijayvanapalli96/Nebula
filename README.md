# Interactive Cinematic Story Engine

This project provides a FastAPI backend for an AI-driven cinematic storytelling experience.

Players start a story with character details, choose actions at each scene, and receive the next scene generated by Gemini with short cinematic prose and structured JSON output.

## What This Application Does

- Starts personalized story sessions with genre, archetype, and motivation.
- Generates scene-by-scene narrative progression using Gemini 1.5 Flash.
- Returns scene data in a strict JSON shape for frontend rendering:
  - scene metadata
  - narrative text
  - visual prompt
  - interactive choices
- Stores session state in memory (MVP), with a repository abstraction ready for Redis migration.
- Adds style seeds to visual prompts (for consistent genre-based image generation quality).

## Tech Stack

- Python 3.11+
- FastAPI
- Uvicorn
- Pydantic v2
- `google-generativeai` SDK

## Project Structure

```text
backend/
  main.py                    # FastAPI app + routes
  schemas.py                 # Pydantic request/response/state models
  story_service.py           # Story flow orchestration
  gemini_story_generator.py  # Gemini client + JSON parsing
  prompts.py                 # Hidden system prompt + style seeds
  state_store.py             # In-memory session repository (Redis-ready surface)
  dependencies.py            # Dependency injection wiring
  config.py                  # Environment-backed settings

  .env.template
```

## Setup

1. Create and activate a virtual environment.
2. Install dependencies.
3. Create your local `.env` from template.
4. Set your Gemini API key.

```bash
cd backend
python -m venv .venv
source .venv/bin/activate
pip install -r requirements.txt
cp .env.template .env
```

Update `.env`:

```env
GEMINI_API_KEY=your_google_ai_studio_api_key
GEMINI_MODEL=gemini-1.5-flash
```

## Run The Service

From the repo root:

```bash
uvicorn main:app --app-dir backend --reload
```

Or from `backend/`:

```bash
python main.py
```

Default server URL:

- `http://127.0.0.1:8000`

Interactive API docs:

- `http://127.0.0.1:8000/docs`

## API Endpoints

### `POST /story/start`

Starts a new story and returns Scene 1.

Request:

```json
{
  "genre": "Noir",
  "name": "Mara Vale",
  "archetype": "Reluctant Detective",
  "motivation": "Find her missing brother"
}
```

Response (shape):

```json
{
  "session_id": "uuid",
  "scene": {
    "metadata": {
      "scene_id": "scene-1",
      "chapter": 1,
      "mood": "suspenseful",
      "tension": 42
    },
    "visual_prompt": "string",
    "narrative_text": "string",
    "choices": [
      {"choice_id": "A", "label": "string", "consequence_hint": "string"},
      {"choice_id": "B", "label": "string", "consequence_hint": "string"},
      {"choice_id": "C", "label": "string", "consequence_hint": "string"}
    ]
  }
}
```

### `POST /story/action`

Sends the selected choice and returns the next scene.

Request:

```json
{
  "session_id": "uuid-from-start",
  "choice_id": "A"
}
```

### `GET /stories/me`

Returns active in-memory story cards for the landing page.

### `GET /health`

Basic health endpoint.

## Implementation Notes

- The backend uses a hidden system prompt to enforce:
  - concise cinematic prose
  - JSON-only output
  - fixed response schema
- Generated visual prompts are enriched with style seeds by genre.
- Session state includes:
  - story config (genre, character, archetype, motivation)
  - history log (scene + player choices)
  - current scene

## Current MVP Limitation

- Session storage is in-memory. Restarting the process clears all sessions.
- To persist sessions in production, replace `InMemoryStoryStateRepository` with a Redis-backed implementation.
